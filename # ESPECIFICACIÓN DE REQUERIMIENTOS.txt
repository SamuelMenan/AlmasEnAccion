# ESPECIFICACIÓN DE REQUERIMIENTOS

# PLATAFORMA DE REGISTRO DE VOLUNTARIADO

```
Versión: 1.
```
## Fecha: 1 de noviembre de 2025

## Resumen:

## Este documento describe los requerimientos funcionales y no funcionales para el desarrollo de la Plataforma de

## Registro de Voluntariado para una ONG. La especificación sigue el estándar IEEE 830 para la definición de

## requisitos de software y cubre el alcance funcional, restricciones técnicas, criterios de aceptación y métricas de

## calidad necesarias para el diseño e implementación del sistema.

## 1.1 GESTIÓN DE USUARIOS Y ROLES

## RF 1 — Registro de voluntarios

### ID VOL-RF-USU- 001

```
Título Registro de voluntarios
```
```
Descripción El sistema debe permitir a nuevos voluntarios registrarse proporcionando datos personales y
validando duplicidad.
```
```
Prioridad Alta
```
```
Categoría Funcional
```
```
Fuente Coordinadores ONG
```
```
Criterios de
Aceptación
```
- Validación de formato de correo.
- Contraseña mínima 8 caracteres con mayúscula y número.
- Campos obligatorios: nombre, apellido, email, teléfono, contraseña.
- Mensaje si el email ya existe.
- Envío de correo de confirmación.

## RF 2 — Autenticación de usuarios

### ID VOL-RF-USU- 002

```
Título Autenticación de usuarios
```
```
Descripción Permite iniciar sesión a voluntarios, coordinadores y administradores con credenciales válidas.
```
```
Prioridad Alta
```
```
Categoría Funcional
```
```
Fuente Administrador ONG
```
```
Criterios de Aceptación - Validar email y contraseña.
```
- Mensaje “Credenciales inválidas” si falla.
- Bloqueo temporal tras 3 intentos.
- Sesión activa 30 min de inactividad.


## RF 3 — Gestión de roles

### ID VOL-RF-USU- 003

```
Título Gestión de roles
```
```
Descripción Permite asignar roles a los usuarios registrados.
```
```
Prioridad Media
```
```
Categoría Funcional
```
```
Fuente Administrador ONG
```
```
Criterios de Aceptación - Solo admins modifican roles.
```
- Todo usuario debe tener rol asignado.
- Mensaje de confirmación al actualizar.
